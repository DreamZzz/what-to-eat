# CLAUDE Memory Index

优先阅读顺序：

1. `TASKS.md`
2. `AGENTS.md`
3. `memory/api-contracts.md`
4. `memory/runtime-modes.md`
5. `memory/deployment.md`
6. `memory/glossary.md`

这个仓库的目标不是维护单一业务，而是维护一套可复制到不同业务的移动端 + 后端脚手架。任何修改都优先问自己：

- 这是平台能力，还是 demo 业务？
- 这项改动会不会让 provider 切换更难？
- 这项改动有没有把模板重新绑回某个真实项目的名字、域名或环境？
